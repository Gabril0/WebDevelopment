Resposta

É possível perceber que este se diferencia no fato de usar Fetch invés do AJAX tradicional, que pega os arquivos JSON retornados pelo fetch e trata as duas possibilidades de erro, a primeira usa um IF e pega o código gerado pela requisição para saber se deu certo, throwing um novo erro caso tenha dado errado, já na segunda trata pelo uso de um bloco catch, não foi utilizada a função de erro passada por parametros.

A implementação do fetch é mais simples quando o usuário entende seu conceito e é bastante poderosa pelo seu encadeamento de tratamento de erros, já a forma tradicional é mais difícil de se implementar e caso haja muitas requisições encadeadas, o código ficaria bastante bagunçado e cheio.
